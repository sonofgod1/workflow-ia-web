# Ideas y features — [Nombre del proyecto]

Captura de ideas que surgieron durante el proyecto pero estaban fuera del scope.
No son compromisos — son semillas para conversaciones futuras.

---

## Cómo usar esta carpeta

Cuando durante una sesión de trabajo surja una idea relevante pero fuera del scope actual:

1. Crear un archivo `YYYY-MM-DD-[slug].md` en esta carpeta
2. Documentar la idea con suficiente contexto para entenderla semanas después
3. No dejar que la idea interrumpa el flujo actual — capturar y seguir

---

## Template de idea

```markdown
# [Título de la idea]
Fecha: YYYY-MM-DD
Surgió durante: /[comando]

## La idea
[Descripción en 2-3 oraciones]

## Por qué puede valer la pena
[Qué problema resuelve o qué oportunidad captura]

## Contexto necesario para evaluarla
[Qué información haría falta para decidir si implementarla]

## Esfuerzo aproximado
[pequeño / mediano / grande]
```
